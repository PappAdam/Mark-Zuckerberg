Installation


The first step is to install Rust. We’ll download Rust through rustup, a command line tool for managing Rust versions and associated tools. You’ll need an internet connection for the download.


Installing rustup on Linux or macOS

$ curl --proto '=https' --tlsv1.3 https://sh.rustup.rs -sSf | sh

Our first code


Now that you’ve installed Rust, let’s write your first Rust program. It’s traditional when learning a new language to write a little program that prints the text Hello, world! to the screen, so we’ll do the same here!


Setting up the project


The first thing we'll do here, is creating a directory for our first rust program. In theory it doesn't matter where your code lives, but its a good practice to keep everything in one place.


Using cargo, we can generate a basic directory:

cargo new *your_dir_name* --bin

If you take a closer look at the file we just got, you should see file named: main.rs

Within that file there is a small program, which should write "Hello World!" into your terminal


You can run the code using:

cargo run

Variables and Mutability


by default variables are immutable. This is one of many nudges Rust gives you to write your code in a way that takes advantage of the safety and easy concurrency that Rust offers. However, you still have the option to make your variables mutable. Let’s explore how and why Rust encourages you to favor immutability and why sometimes you might want to opt out.


When a variable is immutable, once a value is bound to a name, you can’t change that value. To illustrate this lets run our example code!


You can see, that our code raised an error, and our lovely compiler just showed us where to search for the problem. Even it tells us in some cased, that what could be a possible solution.

Here, the problem was, that we created our variable as a immutable u8, since in rust every variable is immutable by default.


We can easily get rid of the problem by passing the mut keyword just before the variable name.


If we run the code again, we should not get any errors.

Constants


Like immutable variables, constants are values that are bound to a name and are not allowed to change, but there are a few differences between constants and variables.

Shadowing


You can declare a new variable with the same name as a previous variable. Rustaceans say that the first variable is shadowed by the second, which means that the second variable is what the compiler will see when you use the name of the variable. In effect, the second variable overshadows the first, taking any uses of the variable name to itself until either it itself is shadowed or the scope ends. Integer Types


An integer is a number without a fractional component. We used one integer type in Chapter 2, the u32 type. This type declaration indicates that the value it’s associated with should be an unsigned integer (signed integer types start with i, instead of u) that takes up 32 bits of space.
